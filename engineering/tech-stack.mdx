## Core Stack

| Layer | Technology | Version |
|-------|-----------|---------|
| **Backend** | Python + Flask | 3.9.18 / 3.0 |
| **Frontend** | Jinja2 + Bootstrap 5 + Vanilla JS | ES6+ |
| **Database** | SQLite | users.db |
| **Data Storage** | JSON files | settings, snapshots, peers |
| **Testing** | pytest | 7.x |
| **CI/CD** | GitHub Actions + Railway | Auto-deploy |
| **Auth** | Google OAuth + bcrypt passwords | Flask sessions |

## Design Principles

1. **Server-rendered** — No SPA framework; Jinja2 templates + vanilla JS for interactivity
2. **Spec-first** — Every feature starts with a specification before implementation
3. **TDD** — Contract tests written before route implementation
4. **JSON-over-DB** — Settings and snapshot data stored as JSON files (no ORM migration overhead)
5. **Blueprint modularity** — FFM and Grants are Flask blueprints; core routes remain in app.py
